项目

FLML (File Layout Markup Language)

背景和目的：

在软件开发工作中，我经常需要解析各种文件。这些文件的格式既有二进制的，也有纯文本的。 为了解析这些文件的结构和内容的涵义，我需要读这些文件的格式说明文档。但是阅读这些文档一方面比较耗费时间，另一方面这些文档的语言性的描述容易产生歧义。 我的目标是创建一种易于使用且准确的标记语言，用于描述二进制文件和纯文本文件的格式。 在设计好这种语言之后，我计划用它来记录生物领域及其他领域中常用的文件格式。该项目的文档位于 file-format.

Gwastools

背景和目的：

在与 GWAS（全基因组关联研究）相关的数据存储中，有几种常见的文件格式，包括 PLINK 文件、BGEN 文件、BAM 文件和 BESD 文件。 然而，执行诸如打印文件基本信息、从文件中提取部分数据、将二进制数据转换为纯文本等操作时，通常需要使用不同的软件。 此外，一些软件在命令行中缺乏帮助信息，一些文件格式也缺少可供编程语言使用的 API。 我的项目有两个主要目标：首先，我希望开发一组 C 语言库，这些库将包含处理这些文件格式所需的核心功能，同时提供 Python API，使用户可以从 Python 中调用这些 C 库； 其次，我计划基于这些库开发一个软件工具，用于更方便地对这些文件执行各类操作。 该项目的代码仓库位于gwastools.

Biotools

背景和目的：

我在 IMCAS 工作期间策划并开展了这个项目。随着工作的推进， 我编写了大量脚本和工具来满足日常工作的需求，其中一些代码具有可重用性。 在这个项目中，我开发了若干用于处理常见生物信息学数据的 Python 类。 此外，我还整理了自己开发的一些实用工具，包括用于系统发育树构建、节点合并与聚类、 长重复序列识别等功能的工具，并将它们加入到了该项目中。该项目的代码托管在 biotools.
